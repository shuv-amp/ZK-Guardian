# Security Policy

## Supported Versions

We support the latest main branch and the most recent tagged release. Older tags are not supported.

## Reporting a Vulnerability

Please report vulnerabilities privately.

- Email: security@zkguardian.io (preferred)
- Include reproduction steps, impact, and any suggested fixes.

We aim to respond within 72 hours. Critical issues are prioritized immediately.

## Security Expectations

- Do not expose secrets in logs.
- Do not weaken authentication checks for production builds.
- ZK circuit artifacts must be verified via checksums before use.
